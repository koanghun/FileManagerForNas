# 프론트엔드 패키지 관리 (npm)

이 문서는 프론트엔드 프로젝트에서 `npm`을 사용하여 패키지를 관리할 때 사용되는 주요 파일인 `package.json`과 `package-lock.json`의 역할에 대해 설명합니다.

## 1. `package.json`

`package.json` 파일은 Node.js 프로젝트의 메타데이터와 의존성 정보를 담고 있는 핵심 파일입니다.

*   **역할:**
    *   **프로젝트 정보:** 프로젝트의 이름, 버전, 설명, 작성자, 라이선스 등 기본 정보를 정의합니다.
    *   **스크립트:** `npm start`, `npm build`, `npm test`와 같이 프로젝트에서 자주 사용되는 스크립트 명령어를 정의합니다.
    *   **의존성 관리 (`dependencies`):** 프로젝트가 실행되는 데 필수적인 라이브러리(예: React, Axios) 목록과 그 버전 범위를 지정합니다.
    *   **개발 의존성 관리 (`devDependencies`):** 개발 및 테스트 단계에서만 필요한 라이브러리(예: 테스트 프레임워크, 번들러, 린터) 목록과 그 버전 범위를 지정합니다.
*   **작동 방식:** `npm install` 명령을 실행하면, npm은 `package.json` 파일의 `dependencies` 및 `devDependencies` 섹션을 읽어 필요한 패키지들을 `node_modules` 폴더에 설치합니다. 버전 범위(예: `^1.0.0`)를 사용하여 최신 호환 버전을 설치할 수 있습니다.

## 2. `package-lock.json`

`package-lock.json` 파일은 `package.json`에 명시된 의존성들이 실제로 어떤 버전으로 설치되었는지에 대한 정확한 정보를 기록하는 파일입니다.

*   **역할:**
    *   **정확한 의존성 트리 기록:** `package.json`이 패키지의 대략적인 버전 범위(예: `^1.0.0`)를 지정하는 반면, `package-lock.json`은 프로젝트에 설치된 모든 패키지(직접적인 의존성뿐만 아니라 의존성의 의존성까지)의 **정확한 버전**을 기록합니다.
    *   **재현 가능한 빌드:** 이 파일 덕분에 다른 개발자가 같은 프로젝트를 설치하거나 CI/CD 환경에서 빌드할 때, 언제 `npm install`을 실행하더라도 **항상 동일한 패키지 버전 구성**을 보장받을 수 있습니다. 이는 "내 컴퓨터에서는 되는데..."와 같은 문제를 방지해 줍니다.
    *   **성능 향상:** `npm install` 명령을 실행할 때 `package-lock.json` 파일이 존재하면, npm은 어떤 패키지를 어떤 버전으로 설치해야 할지 이미 알고 있기 때문에 의존성 해결 과정을 다시 거치지 않아 설치 속도가 빨라질 수 있습니다.
    *   **무결성 검증:** 각 패키지의 `integrity` 해시 값을 포함하여, 설치된 패키지가 변조되지 않았음을 확인할 수 있도록 돕습니다.
*   **작동 방식:** `npm install` 명령을 실행하면 `package-lock.json` 파일이 자동으로 생성되거나 업데이트됩니다. 이 파일은 Git과 같은 버전 관리 시스템에 커밋되어 팀원 간의 일관된 개발 환경을 유지하는 데 사용됩니다.
